PSF library Documentation

Introduction

The goal of the PSF library is to organize molecular models, given in the Protein Data Bank (PDB) format, in order to permit easy and rapid access to structural information. This library has been mainly devised for proteins, and is now extended to all peptides. The current implementation also permits a basic representation of small molecules (e.g. ligands). Our project is to extend it to other types of molecules such as DNA/RNA, and inorganic crystal structures.

PSF objects

The highest-level class in this library is psf_Multimolecule, for description of global environment.

Obviously, molecules are the main objects in the PSF library: psf_Molecule. These generic objects are derived in psf_Peptide, psf_Protein, psf_Ligand objects to describe such types of molecules. In a near future, the package will be extended to other molecule types such as DNA or RNA.

A peptide is a sequence of bonded residues, which form one or several polypeptide chains. A protein can be seen as a complex peptide since, in general, it is composed of a longer sequence of amino acid residues. Besides, a protein may involve several Pchains. The PSF objects corresponding with these structural elements are psf_Residue and psf_Subchain.

The basic elements of a molecule are the atoms, which are the lowest-level class of objects in the PSF library: psf_Atom.

PSF levels

Main API functions

Functions in the PSF API enable access to structural information at the different levels of the hierarchy (e.g. molecule, residue, atom). Functions for constructing the objects are internal to the library.

Installation

The PSF building process relies on cmake utility, available here. Read INSTALL file for more info.

Running the program

A sample program is provided. It runs in a basic terminal, building a set of Psf objects from one or more PDB files, then interpreting several basic commands to explore the content of Psf objects.
