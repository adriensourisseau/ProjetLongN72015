AMC library Documentation

Introduction

The AMC library permits to represent and manipulate molecules as articulated mechanisms. Groups of rigidly bonded atoms form the bodies of the articulated molecular chains. This library provides a set of functions to efficiently access and modify the spatial location of such structures. Two types of motion can be performed: ensemble motions (translation, rotation) of molecules or sub-chains, and rotation motions (bond torsions) between adjacent bodies.

AMC objects are constructed from the corresponding PSF objects (see PSF Documentation) and complementary information contained in .aaa and .jnt files (see below for details about AAA and JNT formats).

AMC objects

The highest-level class in this library is amc_Multimolecule. This is the equivalent class to the psf_Multimolecule in the PSF library. Each molecule in PSF also has its corresponding mechanical representation in AMC: amc_Molecule. (In the current version there is no differentiation between molecule types - protein, ligand - at the mechanical level).

Molecules are divided into sub-chains (amc_Subchain). They may correspond to chemical sub-chains (e.g. the different chains of a multimeric protein), or to a mechanical decomposition of the molecule for permitting independent motions of segments of a single molecular chain. Let us consider for instance a small protein composed by two helices and a loop between them. Generating independent motions of the two helices will require to divide the molecular chain into (at least) two mechanical sub-chains containing each of them. Bonds between atoms in different sub-chains will be maintained using amc_Constraints (these objects are not available in the current version).

AMC representation of a protein segment

Objects are also associated with the main-chain and the side-chains of these mechanical tree-like sub-chain: amc_Mainchain, amc_Sidechain.

Rigidly linked groups of atoms (amc_Atom) form the rigid bodies of the mechanism: amc_Rigid. These rigid bodies may have very different size. For instance, a simplified modeling of proteins may consider secondary structure elements (helices and sheets) to be rigid. The flexible loop is however composed of many small bodies. The figure below illustrates how a pair of residues are decomposed into rigid bodies when side-chain flexibility is considered.

The connectivity of rigids is described by the object class amc_RigidTree associated with each amc_Rigid.

Consecutive rigid bodies in an articulated molecular chain are connected by revolute joints (corresponding with bond torsions). The class amc_Dihedral represents such joints. Each object amc_Rigid contains an amc_Dihedral.

For enabling global motions of sub-chains and molecules, free-flying joints (amc_Freefly) are associated with theses classes of objects. The classes amc_Dihedral and amc_Freefly are specializations of the generic class amc_Joint.

Main API functions

Functions in the AMC API enable access to information at the different levels of the mechanistic molecular model, and to modify its spatial location. The more useful functions are change_joint_value (only for rigids), change_location, update_location, and move, which are member functions of the main object classes. Constructing functions, although accessible, are only for internal use (they are not commented in this documentation).

Joint files

Description of joints needs an independent file, allowing the user to specify flexibility of molecule. For a peptide or a protein, flexibility is described at backbone or sidechain level. For a ligand, mobile chemical bonds are described by the atoms involved in the bond.

AMC file

This format is the common format for all kinds of molecules. Each line contains a keyword, and parameters depending on keyword :

  • Molecule defining type (protein/peptide/ligand) of molecule
  • Residue to describe backbone and sidechain flexibility for each residue. Parameters are coefficients of flexibility (probability to vary while we sample joints values, 0.0 for rigid, 1.0 for full flexibility).
  • Tripeptide to break a polypeptide chain into 3 residues subchain.
  • Joint to describe flexibility of a ligand, with the 4 atoms specifying the dihedral, and 1 to 3 pairs of maximal angles.
  • Subchain to break a polypeptide chain into several mechanical subchains.

A template amc file can be generated for a peptide or a protein by the build_amc tool : build_amc [<options>] <name>.pdb

Installation

The Psf-Amc building process relies on cmake utility, available here. Read INSTALL file for more info.

Running the program

A sample program is provided. It runs in a basic terminal, building a set of Psf and Amc objects from one or more PDB files and AMC associated files, interpreting several basic commands to explore the content of Psf and Amc objects.
